# SEO Report for yangmingli.com

## Summary

The repository is a static HTML site. There is no Next.js, React build app, Astro, Gatsby, WordPress, or package-based frontend framework in this repo. Public routes are generated from physical `.html` files and folder `index.html` files. The sitemap is generated by `generate_sitemap.py`.

Key SEO issues found:

- The site relied heavily on homepage tab content, leaving important sections like About, Projects, Blog, Resume, and Contact without dedicated crawlable URLs.
- Several article pages had duplicate canonical tags.
- Some pages had weak or truncated meta descriptions such as `By Yangming Li...`.
- The homepage pointed to a missing `/en/` hreflang URL and a missing `/manifest.json`.
- `product/index.html` was a duplicate article index with mojibake text and broken internal links to `/aiml/`, `/playground/`, and `/contact/`.
- Two pages referenced missing Prism assets.
- `machine-unlearning-guide.html` referenced a missing diagram image.
- Old Search Console sitemap URLs, `/sitemap/google.xml` and `/sitemap.txt`, were not valid sitemap files.
- A zero-byte spam-looking legacy path exists under `/nha-cai/`; it is not included in the sitemap and has no internal links.

## Framework and Routing

- Framework: static HTML.
- Routes: root `.html` files, nested `.html` files, and folder `index.html` routes.
- Metadata: mostly hard-coded in each HTML file.
- Sitemap: generated by `generate_sitemap.py`.
- Robots: served from `/robots.txt`.
- Canonicals: hard-coded per page and now normalized to `https://yangmingli.com`.
- Structured data: JSON-LD in page heads/bodies; syntax validated locally.
