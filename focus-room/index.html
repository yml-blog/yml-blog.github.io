Architecture

Clean SwiftUI structure with reusable parts

The prototype is organized as a lightweight SwiftUI app with a single orchestrating view model, focused services for persistence and audio, and reusable presentation components that keep the code modular without turning the experience into a dashboard.

1. App Architecture

AppState + FocusRoomViewModel

AppState owns the feature, while FocusRoomViewModel coordinates entry, timer state, ghost UI fading, ambient layers, completion, and persistence.

  • Single source of truth for phase, timer, mixer, and room atmosphere
  • Persistence isolated behind a FocusRoomPreferencesStoring protocol
  • Mock audio engine keeps the first prototype shippable without real audio assets
2. View Structure

Views stay focused and composable

The UI is decomposed into exactly the kinds of pieces you would want to lift into an actual Xcode project: threshold, room, timer, mixer, background, completion, and a small ghost-glass wrapper.

3. Models and State

Session state stays explicit

The model layer separates user preference data from runtime session state, so the prototype can remember the room someone set up without persisting transient countdown behavior.

  • AmbientLayerKind and AmbientLayerSetting for the five sound layers
  • FocusRoomPreferences for last used mixer settings and duration
  • RoomAtmosphere as a derived model for lamp warmth, rain intensity, and visual depth
4. Persistence and Placeholders

Ready to evolve without login

The prototype stores preferences locally using UserDefaults. The audio layer is intentionally mocked so the visual and interaction quality can be refined before real assets are introduced.

  • Bundled preview data for SwiftUI previews
  • In-memory store for local previews and design iteration
  • Clear seam for swapping mock audio with real fade-aware playback later
